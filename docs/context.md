# context.md — 次回セッション引き継ぎ用

## 監査ログ基盤（方針決定済み・未実装）

### 課題
- IPO準備に向けて「誰がいつ何をしたか」の記録が必要
- 現在はログが一切ないため、データ変更の追跡ができない

### 決まった方針

**記録するもの：**
- 認証（ログイン、ログアウト、ログイン失敗）
- データ変更（エンジニア・案件・会社情報の追加・編集・削除）
- 管理者操作（アカウント作成・削除、ロール変更、パスワードリセット）
- システム操作（メール取得、AI解析、CSVインポート）
- 設定変更

**記録しないもの：**
- 閲覧ログ（誰が何を見たか）
- 操作ログ（ページ遷移など）

**1件あたりの記録内容：**
- いつ（日時）
- 誰が（ユーザーID・名前）
- 何をした（操作の種類）
- 何に対して（対象データ）
- 変更内容（変更前→変更後）

### 実装タイミング
- 会社のインフラ整備（Render/Supabase/Vercelを会社アカウントに移行）と同時に導入
- 現在は個人アカウントで運用中のため、移行が先

### 未決事項
- ログの保存先（Supabase内のテーブル？外部サービス？）
- ログの保持期間
- ログの閲覧UI（管理画面に組み込むか）
- 具体的なテーブル設計・実装方式（実装時に /ask-codex で設計レビュー予定）
